Debug log - April 12

Ran pipeline on GunPoint dataset.
Expected accuracy ~0.85, got 0.0 for both raw and catch22.

Hypothesis: label encoding is wrong.
Checked: y values are [130, 15, 135, 94, 34, ...] — these are NOT class labels!
They look like row indices being used as labels.

Root cause: the .ts file uses colon-separated labels (e.g. "...,0.64:2")
but our parser splits on comma and takes the last element, which gives
a garbage value from the last float in the data row.

Fix: use rsplit(':', 1) to correctly separate data from label.

Fix verified - April 12 (11:00)
After applying rsplit fix:
  GunPoint  raw=0.8267  catch22=0.8867  (was 0.0 before fix)
  ECG200    raw=0.8100  catch22=0.7700
Label parsing is now correct.

Data leakage bug - April 14 (14:00)
MSE was suspiciously low (0.001) on ETTh1.
Found: target column 'OT' was accidentally included in the feature matrix.
Model was essentially predicting OT from OT — trivial task.
Fix: explicitly drop 'OT' from X before training.

MLP convergence issue - April 15 (10:30)
MLPRegressor not converging — ConvergenceWarning after 80 iterations.
Embeddings are noisy and MSE is worse than lag+rolling baseline.
Root cause: default learning rate (0.001) too slow for 80 epochs.
Also: no early stopping, so training stops at max_iter regardless.
Fix: add early_stopping=True and increase n_iter_no_change.

SHAP pruning on forecasting - April 15 (16:45)
Applied SHAP to lag+rolling features (28 total).
SHAP removed 12 features (43% reduction).
MSE with pruned features: 0.2303 vs 0.2283 without pruning.
Negligible difference — confirms many lag features are redundant.

Catch22 anomaly detection - April 16 (13:30)
Applied catch22 to PPG windows instead of raw windows.
Results:
  Raw windows:  F1=0.871, AUC=0.996
  Catch22:      F1=0.581, AUC=0.938  <-- much worse!

Why? Catch22 normalizes amplitude-based features.
Spike anomalies are defined by high amplitude, but catch22 features
like z-score outlier fraction get normalized away.
Investigating SHAP-guided selection to recover performance.

SHAP-guided anomaly detection - April 17 (09:30)
Used supervised LightGBM proxy to get SHAP importances on catch22 features.
Top features: diff_variance, lempel_ziv, time_reversibility, nonlinearity
Removed: hist_entropy, spectral_entropy, peak_freq (these were hurting)
Results after SHAP selection (12/22 features kept):
  Catch22+SHAP: F1=0.710, AUC=0.959  (was 0.581 with all catch22 features)
SHAP-guided selection successfully recovers most of the performance.

Final summary - April 27
All bugs fixed. Pipeline runs cleanly end-to-end.
