\begin{abstract}
Extracting meaningful features from time-series data remains a critical bottleneck in building effective machine learning models. While deep learning approaches offer powerful end-to-end solutions, they are often computationally expensive and lack interpretability. Building upon our Phase 1 proposal, this project implements and evaluates a unified framework combining hybrid, automated, and explainable feature engineering techniques across three distinct time-series tasks: classification, forecasting, and anomaly detection. We evaluate the Catch22 feature set, lag and rolling statistics, and self-supervised MLP embeddings, coupled with SHAP-guided feature pruning. Our extensive experiments on the UCR Archive, ETTh1, and a simulated WESAD-like dataset demonstrate that hybrid models (e.g., Catch22 with LightGBM) can achieve comparable or superior performance to raw baselines while significantly reducing dimensionality. Furthermore, SHAP-guided pruning successfully reduced feature spaces by up to 50\% without compromising accuracy, highlighting the efficacy of explainability in automated feature selection.
\end{abstract}

\section{Introduction}
\IEEEPARstart{T}{ime-series} data is ubiquitous across domains ranging from healthcare to energy management. Unlike static tabular data, time-series possess inherent temporal dependencies, making direct application of standard machine learning algorithms suboptimal. Feature engineering---the process of transforming raw sequences into informative characteristics---is essential for capturing trends, seasonality, and volatility.

In our initial proposal, we identified a growing trend towards hybrid systems that combine handcrafted features with lightweight models, as well as the integration of Explainable AI (XAI) into the feature selection process. Deep learning models, while powerful, often act as ``black boxes'' and demand significant computational resources. Conversely, traditional automated feature engineering (AutoFE) tools can generate thousands of redundant features, leading to the curse of dimensionality. 

This project bridges these gaps by implementing a comprehensive, multi-task evaluation framework. We address the feedback from our Phase 1 proposal by clearly defining baselines, separating evaluations by task, and demonstrating how SHAP (SHapley Additive exPlanations) values can actively guide modeling decisions rather than merely serving as post-hoc explanations. 

Our core objectives are:
\begin{enumerate}
    \item To evaluate the effectiveness of the Catch22 feature set against raw temporal data for classification.
    \item To compare traditional lag/rolling statistics against self-supervised embeddings for multivariate forecasting.
    \item To demonstrate the utility of SHAP-guided feature pruning in reducing computational overhead while maintaining model performance.
\end{enumerate}

\section{Literature Review}
Our methodology is grounded in recent advancements in time-series feature engineering. We build upon the concept of hybrid models introduced by Badi et al. (2024) \cite{cocalite}, who demonstrated that combining Catch22 features with a lightweight neural network (COCALITE) could rival massive architectures like InceptionTime while requiring 95\% fewer parameters. 

For spatial and multivariate forecasting, D'Aversa et al. (2024) \cite{split} highlighted the importance of localized feature representations through Spatio-Temporal Linear Model Trees (SPLiT). Similarly, Okadome and Nakamura (2024) \cite{lag_op} showed that self-supervised lag-prediction tasks could force neural networks to learn meaningful interaction features without explicit labels.

In the domain of unsupervised anomaly detection, Valerio et al. (2024) \cite{ppg_anom} successfully utilized Catch22 features with Isolation Forests to detect artifacts in wearable PPG data, proving that robust features can mitigate the need for labeled training data.

Finally, our feature selection strategy is inspired by Petrosian et al. (2025) \cite{autofe_xai}, who proposed AutoFE-XAI, a framework utilizing SHAP values to prune redundant features. We extend this idea by applying SHAP pruning across classification, forecasting, and anomaly detection tasks.

\subsection{Datasets}
\begin{itemize}
    \item \textbf{UCR Time Series Archive:} Used for classification. We selected five diverse datasets (GunPoint, ECG200, ItalyPowerDemand, SyntheticControl, TwoLeadECG) to test cross-domain generalization.
    \item \textbf{ETTh1 (Electricity Transformer):} Used for multivariate forecasting. It contains hourly data of 7 variables, targeting Oil Temperature (OT).
    \item \textbf{WESAD-like PPG Data:} A simulated dataset modeling wearable blood volume pulse (BVP) signals with injected anomalies (spikes, flatlines, noise) to evaluate unsupervised anomaly detection.
\end{itemize}

\subsection{Feature Engineering Techniques}
\textbf{1. Catch22 Features:} We implemented the 22 canonical features from the Catch22 library \cite{catch22}, which capture diverse time-series properties including autocorrelation, entropy, and scaling. 

\textbf{2. Lag and Rolling Statistics:} For forecasting, we generated lag features ($t-1, t-2, \dots$) and rolling window statistics (mean, standard deviation, min, max) to capture local temporal dynamics.

\textbf{3. Self-Supervised Embeddings:} We trained a Multi-Layer Perceptron (MLP) to predict the current value based on a window of past values. The hidden layer activations were extracted as dense feature embeddings.

\textbf{4. SHAP-Guided Pruning:} We utilized LightGBM models as proxy evaluators. By computing the mean absolute SHAP values for all generated features, we pruned the bottom 40-45\% of features, retaining only the most impactful predictors.

\subsection{Evaluation Models and Baselines}
To quantify the gains from feature engineering, we established clear baselines for each task:
\begin{itemize}
    \item \textbf{Classification:} Baseline = LightGBM on raw time-series data. Proposed = LightGBM on Catch22 features.
    \item \textbf{Forecasting:} Baseline = Ridge Regression on lag-1 data. Proposed = LightGBM on Lag+Rolling and Embeddings.
    \item \textbf{Anomaly Detection:} Baseline = Isolation Forest on raw data windows. Proposed = Isolation Forest on Catch22 features.
\end{itemize}

\section{Results and Analysis}

\subsection{Task 1: Time-Series Classification}
We evaluated the models across five UCR datasets. Figure \ref{fig:clf_acc} displays the accuracy comparison.

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{figures/fig1_classification_accuracy.png}
    \caption{Classification Accuracy across 5 UCR datasets comparing Raw Features, Catch22, and Catch22+SHAP.}
    \label{fig:clf_acc}
\end{figure}

The Catch22 features demonstrated strong performance, particularly on the \textit{SyntheticControl} and \textit{GunPoint} datasets, where they outperformed the raw baseline. On \textit{SyntheticControl}, accuracy improved from 0.886 to 0.950. However, on datasets with highly specific local shape dependencies like \textit{ItalyPowerDemand}, raw features maintained an edge. Crucially, applying SHAP-guided pruning (reducing the feature set from 22 to 13) resulted in virtually identical accuracy and F1-scores, proving that many Catch22 features are redundant for specific tasks.

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{figures/fig3_shap_importance.png}
    \caption{SHAP Feature Importance for the GunPoint dataset. The top features drive the pruning process.}
    \label{fig:shap_gunpoint}
\end{figure}

Figure \ref{fig:shap_gunpoint} illustrates how SHAP values guide the modeling decisions. For the GunPoint dataset, features related to autocorrelation and distribution spread were heavily favored, allowing us to confidently discard entropy-based features without performance loss.

\subsection{Task 2: Multivariate Forecasting}
For the ETTh1 dataset, the goal was to forecast Oil Temperature (OT). The results are summarized in Figure \ref{fig:forecasting_metrics}.

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{figures/fig4_forecasting_metrics.png}
    \caption{Forecasting MSE and MAE on the ETTh1 dataset.}
    \label{fig:forecasting_metrics}
\end{figure}

The simple lag-1 baseline yielded a high Mean Squared Error (MSE) of 1.2856. Introducing Lag and Rolling statistics drastically reduced the MSE to 0.2283. The self-supervised MLP embeddings performed reasonably well (MSE 0.3370) but did not surpass the explicit statistical features. Applying SHAP pruning to the Lag+Rolling set removed 12 of the 28 features while maintaining an MSE of 0.2303, demonstrating significant computational efficiency gains for inference.

\subsection{Task 3: Anomaly Detection}
Using the simulated WESAD-like PPG data, we attempted to isolate anomalous signal segments using an unsupervised Isolation Forest. 

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{figures/fig7_anomaly_metrics.png}
    \caption{F1-Score and ROC-AUC for Anomaly Detection.}
    \label{fig:anom_metrics}
\end{figure}

Interestingly, the raw window baseline performed exceptionally well (F1=0.8710) because the injected anomalies (spikes and flatlines) drastically altered the raw amplitude. Catch22 alone struggled slightly (F1=0.5806), likely due to the features normalizing out some of the absolute amplitude differences. However, when we applied SHAP-guided pruning (using a proxy classifier to identify the most descriptive Catch22 features for anomalies), the F1-score rebounded to 0.7097.

\section{Discussion and Conclusion}

\subsection{Generalization Limits}
While Catch22 provides an excellent, lightweight baseline, our results on \textit{ItalyPowerDemand} highlight cross-domain generalization limits. Global statistical features struggle when the target class is defined by a very brief, specific local pattern (e.g., a sudden drop at a specific timestamp). In such cases, raw features or localized embeddings are necessary.

\subsection{The Value of Explainability}
Our implementation successfully demonstrated that XAI (SHAP) is not just a post-hoc reporting tool. By actively using SHAP to prune features, we reduced the dimensionality of our models by an average of 45\% across all tasks. This translates directly to faster training times and lower inference costs, which is critical for deployment on edge devices or wearable sensors.

\subsection{Conclusion}
This project successfully implemented and evaluated a hybrid, automated, and interpretable feature engineering framework. We showed that smart feature engineering (Lag+Rolling, Catch22) combined with gradient boosting (LightGBM) can solve complex forecasting and classification tasks efficiently. Furthermore, automated SHAP pruning proved to be a highly reliable method for optimizing feature sets. Future work will focus on integrating Large Language Models (LLMs) to automatically suggest domain-specific features based on dataset metadata prior to empirical evaluation.
