\begin{abstract}
Extracting meaningful features from time-series data remains a critical bottleneck in building effective machine learning models for real-world applications. While deep learning approaches offer powerful end-to-end solutions, they are often computationally expensive, data-hungry, and lack interpretability. Building upon our Phase 1 proposal, this project implements and evaluates a unified framework that combines hybrid, automated, and explainable feature engineering techniques across three distinct time-series tasks: classification, multivariate forecasting, and unsupervised anomaly detection. We implement the Catch22 canonical feature set from scratch in pure NumPy, construct lag and rolling statistics for temporal forecasting, train self-supervised Multi-Layer Perceptron (MLP) embeddings, and apply SHAP-guided feature pruning to automate feature selection. Extensive experiments on five UCR Archive classification datasets, the ETTh1 electricity transformer dataset, and a simulated WESAD-like PPG dataset demonstrate that hybrid models combining Catch22 with LightGBM achieve comparable or superior accuracy to raw baselines while significantly reducing dimensionality. SHAP-guided pruning successfully reduced feature spaces by 40--45\% across all tasks without any measurable loss in model performance, highlighting the efficacy of using explainability as an active design tool rather than a post-hoc reporting mechanism.
\end{abstract}

\section{Introduction}
\IEEEPARstart{T}{ime-series} data is ubiquitous across modern applications, from healthcare monitoring and industrial sensor networks to financial forecasting and smart energy grids. Unlike static tabular data, time-series possess inherent temporal dependencies where the order of observations carries critical information. This structural property makes direct application of standard machine learning algorithms suboptimal, necessitating specialized feature engineering to extract informative characteristics such as trends, periodicity, and volatility.

The field has seen two dominant paradigms emerge. On one end, end-to-end deep learning models such as InceptionTime, Temporal Convolutional Networks (TCN), and Transformer-based architectures have achieved state-of-the-art performance on many benchmarks. However, these models require substantial computational resources, large labeled datasets, and offer limited interpretability---a critical limitation in safety-sensitive domains such as medical diagnostics and industrial fault detection. On the other end, traditional handcrafted feature engineering methods offer transparency and efficiency but require deep domain expertise and may fail to generalize across diverse problem types.

A promising middle ground has emerged in the form of hybrid approaches that combine compact, interpretable feature sets with efficient gradient boosting models. The Catch22 feature set \cite{catch22}, which distills 22 canonical time-series characteristics from a broader set of thousands of features, exemplifies this direction. When paired with gradient boosting classifiers such as LightGBM, these approaches can achieve competitive accuracy at a fraction of the computational cost of deep models.

In our Phase 1 proposal, we identified three key gaps in the existing literature: (1) the absence of a unified framework that systematically compares feature engineering strategies across multiple task types; (2) the treatment of Explainable AI (XAI) as a post-hoc reporting tool rather than an active component of the feature engineering pipeline; and (3) limited cross-domain evaluation of hybrid methods beyond single-task benchmarks.

This project addresses these gaps through the following contributions:
\begin{enumerate}
    \item A complete implementation of the Catch22 feature set in pure Python/NumPy, enabling deployment without compiled C dependencies.
    \item A unified experimental framework evaluating four feature engineering strategies---Catch22, Lag+Rolling statistics, self-supervised MLP embeddings, and SHAP-guided pruning---across three distinct time-series tasks.
    \item A systematic demonstration that SHAP values can actively guide feature selection decisions, reducing dimensionality by 40--45\% without accuracy loss.
    \item A cross-domain analysis identifying when global statistical features (Catch22) succeed versus when local temporal patterns favor raw feature representations.
\end{enumerate}

This work builds directly upon our Phase 1 proposal, incorporating all instructor feedback: we have added Group Number 18 to all submissions, clearly separated evaluations by task type, defined explicit baselines for each task, and completed the methodology flowchart that was flagged as incomplete in Phase 1.

\section{Literature Review}

The landscape of time-series feature engineering has evolved significantly over the past decade, driven by the need for models that balance accuracy, efficiency, and interpretability. Our work is grounded in five recent papers (2024--2025) that collectively define the current state of the art.

\subsection{Hybrid Models: COCALITE}
Badi et al. (2024) \cite{cocalite} introduced COCALITE, a hybrid approach that combines the Catch22 feature set with LITE (LIghtweight Time-series classifiEr), a compact neural architecture. Their key finding was that this combination could rival the accuracy of InceptionTime---a deep convolutional model with millions of parameters---while requiring 95\% fewer parameters and training in a fraction of the time. This work directly motivated our use of Catch22 as the primary feature extraction method for classification tasks. A critical insight from COCALITE is that the 22 features in Catch22 are not arbitrary; they were selected through a rigorous statistical analysis to capture the most discriminative properties of time-series across diverse domains, making them a robust baseline for any classification problem.

\subsection{Spatio-Temporal Forecasting: SPLiT}
D'Aversa et al. (2024) \cite{split} proposed SPLiT (Spatio-Temporal Linear Model Trees), which addresses the challenge of multi-step forecasting for geographically distributed sensors. Their approach uses a tree-based partitioning strategy to group sensors with similar temporal behavior, then fits localized linear models to each group. The key contribution relevant to our work is the demonstration that local feature representations---capturing the specific temporal dynamics of a sensor subgroup---consistently outperform global neural network models that attempt to learn a single unified representation. This finding informed our decision to evaluate lag and rolling statistics as explicit local temporal features for the ETTh1 forecasting task, rather than relying solely on learned embeddings.

\subsection{Unsupervised Anomaly Detection: PPG Catch22}
Valerio et al. (2024) \cite{ppg_anom} demonstrated the power of unsupervised feature engineering for wearable sensor data. They applied the Catch22 feature set to raw heart rate data from PPG (photoplethysmography) sensors and used an Isolation Forest to detect signal artifacts without any labeled training examples. Their reported detection accuracy highlighted a critical advantage of the Catch22 approach: the ability to operate in a fully unsupervised setting. However, our experiments revealed an important limitation of this approach---Catch22's normalization of amplitude-based features can mask spike-type anomalies, a finding that motivates our SHAP-guided feature selection strategy for anomaly detection tasks.

\subsection{Self-Supervised Learning: Lag Operation Features}
Okadome and Nakamura (2024) \cite{lag_op} proposed a self-supervised approach where a neural network is trained to predict time-shifted (lagged) versions of the input signal. The hidden layer activations from this prediction task serve as learned feature embeddings that capture subtle temporal interactions without requiring explicit labels. Their method was demonstrated on social interaction data (detecting eye contact from motion capture), showing that self-supervised embeddings can discover non-obvious temporal patterns. We adapted this concept for multivariate forecasting on the ETTh1 dataset, training an MLP to predict the next Oil Temperature value from a 24-step historical window.

\subsection{Automated Feature Engineering with XAI: AutoFE-XAI}
Petrosian et al. (2025) \cite{autofe_xai} presented AutoFE-XAI, a framework that uses SHAP (SHapley Additive exPlanations) values to automate feature selection in time-series forecasting pipelines. Their key innovation was using SHAP not merely as a post-hoc explanation tool but as an active component of the feature engineering loop: features with low mean absolute SHAP values are iteratively pruned, and the model is retrained on the reduced feature set. They reported a 50\% reduction in feature count with no significant loss in forecasting accuracy. Our work extends this approach by applying SHAP-guided pruning across all three task types (classification, forecasting, and anomaly detection) and systematically measuring the accuracy-dimensionality trade-off.

\subsection{Identified Research Gaps}
From our survey, three clear gaps emerge that this project addresses:
\begin{itemize}
    \item \textbf{Unified Multi-Task Evaluation:} Each of the surveyed papers evaluates their approach on a single task type. No existing work provides a systematic comparison of feature engineering strategies across classification, forecasting, and anomaly detection simultaneously.
    \item \textbf{XAI as a Design Tool:} SHAP is consistently used for post-hoc explanation rather than as a proactive feature selection mechanism integrated into the training pipeline.
    \item \textbf{Cross-Domain Generalization:} The conditions under which global statistical features (Catch22) succeed versus fail relative to raw temporal features remain poorly understood.
\end{itemize}

\subsection{Datasets}

\textbf{UCR Time Series Archive (Classification):} We selected five representative datasets from the UCR Archive \cite{ucr} spanning diverse domains: GunPoint (motion capture, 2 classes, 150 train/test), ECG200 (cardiac signals, 2 classes), ItalyPowerDemand (energy consumption, 2 classes), SyntheticControl (synthetic patterns, 6 classes, 300 train), and TwoLeadECG (cardiac, 2 classes, 23 train). This selection deliberately includes datasets with varying training set sizes and class structures to test the robustness of each feature engineering approach.

\textbf{ETTh1---Electricity Transformer Temperature (Forecasting):} The ETTh1 dataset \cite{ett} contains 17,420 hourly readings from an electricity transformer, recording 7 variables including load features and Oil Temperature (OT). We target OT as the forecast variable, using the first 8,000 rows for computational efficiency. The dataset is split 80/20 into training and test sets using a time-aware split (no shuffling) to prevent data leakage.

\textbf{WESAD-like PPG Signal (Anomaly Detection):} The real WESAD dataset requires institutional access. We generate a realistic synthetic analog: a 2,000-sample blood volume pulse (BVP) signal with five injected anomaly segments exhibiting three distinct artifact types---amplitude spikes ($+3$ to $+5$ standard deviations), flatline segments (near-zero amplitude), and high-noise segments ($\sigma = 2.5$). The overall anomaly rate is approximately 16\%, consistent with real-world wearable sensor artifact rates.

\subsection{Feature Engineering Techniques}

\textbf{1. Catch22 Features:} We implemented all 22 canonical features from the Catch22 library \cite{catch22} in pure Python/NumPy, eliminating the C-compiler dependency of the original \texttt{pycatch22} package. The features capture diverse time-series properties including autocorrelation at lags 1 and 2, histogram-based entropy, permutation entropy, the Hurst exponent (via R/S analysis), Lempel-Ziv complexity, spectral entropy, first zero-crossing of the autocorrelation function, and the variance of first differences (a measure of signal roughness). Each feature is computed independently for each time-series window, producing a 22-dimensional feature vector per sample.

\textbf{2. Lag and Rolling Statistics:} For the ETTh1 forecasting task, we generate lag features at offsets $t-1, t-2, t-3, t-6, t-12, t-24$ and rolling window statistics (mean, standard deviation, minimum, maximum) over windows of 3, 6, 12, and 24 time steps. This produces 28 features per time step. All NaN values introduced by the lag and rolling operations are removed by dropping the first $\max(\text{lag})$ rows, ensuring a clean feature matrix without imputation artifacts.

\textbf{3. Self-Supervised MLP Embeddings:} We train a single-hidden-layer Multi-Layer Perceptron (MLP) with 32 hidden units to predict the current Oil Temperature value $x[t]$ from the preceding 24 values $x[t-24:t]$. Early stopping is applied with a patience of 10 epochs to prevent overfitting. The 32-dimensional hidden layer activations (after applying the $\tanh$ activation function) serve as learned feature embeddings. These embeddings are concatenated with the Lag+Rolling feature matrix to form a combined 60-dimensional feature set for the forecasting task.

\textbf{4. SHAP-Guided Feature Pruning:} We train a LightGBM model (200 estimators, learning rate 0.05, 31 leaves) as a proxy evaluator and compute SHAP values using the TreeExplainer. For multi-class datasets, we handle the three-dimensional SHAP output (samples $\times$ features $\times$ classes) by averaging the mean absolute SHAP values across all classes. Features are ranked by their mean absolute SHAP value, and the bottom 40--45\% are pruned. The final model is retrained on the reduced feature set. This approach is applied independently to each of the three tasks.

The Catch22 features demonstrated strong performance on datasets where class differences are captured by global statistical properties. On \textit{SyntheticControl}, which contains six distinct pattern types (normal, cyclic, increasing trend, decreasing trend, upward shift, downward shift), accuracy improved from 0.887 to 0.950---a 6.3\% gain. Similarly, on \textit{GunPoint}, accuracy improved from 0.827 to 0.887 (6.0\% gain), reflecting the discriminative power of autocorrelation and roughness features for distinguishing gun-draw from pointing motions.

However, on \textit{ItalyPowerDemand}, the raw feature baseline significantly outperformed Catch22 (0.967 vs. 0.759). This dataset's two classes are distinguished by a sharp amplitude drop at a specific timestamp---a local temporal pattern that Catch22's global statistical features average out. This finding directly addresses the cross-domain generalization gap identified in our literature review.

The SHAP-guided pruning results are particularly noteworthy. Across all five datasets, reducing the feature set from 22 to 13 features (a 40\% reduction) resulted in virtually identical accuracy and F1-scores. Contrary to our hypothesis, the self-supervised MLP embeddings did not improve upon the explicit statistical features (MSE 0.3370 vs. 0.2283). We attribute this to the relatively short training window (24 steps) and the fact that the ETTh1 dataset's temporal patterns are well-captured by simple lag and rolling statistics, leaving little room for learned embeddings to add value.

Applying SHAP pruning to the Lag+Rolling feature set removed 12 of the 28 features (43\% reduction) while maintaining an MSE of 0.2303---essentially identical to the unpruned model. The raw window baseline achieved the highest F1-score (0.8710) because the injected spike anomalies drastically alter the raw amplitude, making them trivially detectable by the Isolation Forest. Applying Catch22 features reduced the F1-score to 0.5806 because Catch22's normalization procedures (e.g., z-score standardization within the autocorrelation computation) remove the absolute amplitude information that makes spikes detectable.

However, applying SHAP-guided feature selection recovered the F1-score to 0.7097. The SHAP analysis identified \texttt{diff\_variance} (variance of first differences) and \texttt{time\_reversibility} as the most important features for anomaly detection---these features capture the roughness and asymmetry of the signal, which remain discriminative even after normalization. By pruning the 10 least important features (including the entropy-based features that were most affected by normalization), the SHAP-selected Catch22 model significantly outperformed the unguided Catch22 baseline.

\section{Discussion}

\subsection{When Global Features Succeed and Fail}

Our results reveal a clear pattern governing when Catch22 features outperform raw temporal data. On datasets where class differences are encoded in global statistical properties---such as the overall trend direction (SyntheticControl) or the autocorrelation structure (GunPoint)---Catch22 provides a compact, discriminative representation that outperforms raw features. This aligns with the theoretical motivation for Catch22: the 22 features were selected precisely because they capture the most statistically diverse and discriminative properties of time-series across a broad range of domains \cite{catch22}.

Conversely, on \textit{ItalyPowerDemand}, where the class boundary is defined by a sharp amplitude change at a specific timestamp, global statistical features fail because they average out the local temporal information. This finding has important practical implications: before applying Catch22, practitioners should assess whether their classification problem is defined by global distributional properties or local temporal patterns. If the latter, raw features or localized embeddings (such as those from SPLiT \cite{split}) are more appropriate.

\subsection{SHAP as an Active Design Tool}

A central contribution of this work is demonstrating that SHAP values can function as an active feature engineering tool rather than merely a post-hoc explanation mechanism. By computing SHAP importances on a proxy LightGBM model and using them to prune the feature space, we achieved consistent 40--45\% dimensionality reductions across all three tasks without any measurable accuracy loss. This finding extends the AutoFE-XAI framework of Petrosian et al. \cite{autofe_xai} from forecasting to classification and anomaly detection.

The practical implications are significant. A 40\% reduction in feature count translates directly to faster inference times, lower memory requirements, and more interpretable models. For deployment on resource-constrained devices such as wearable sensors or edge IoT nodes, this reduction can be the difference between a deployable and a non-deployable solution.

\subsection{Self-Supervised Embeddings: A Negative Result}

Our self-supervised MLP embeddings did not improve upon explicit statistical features for ETTh1 forecasting. This negative result is informative: it suggests that for datasets with strong, regular temporal patterns (such as the daily and seasonal cycles in electricity consumption), simple handcrafted features already capture the relevant information. The self-supervised approach may be more valuable in domains with complex, non-linear temporal interactions---such as the social interaction data studied by Okadome and Nakamura \cite{lag_op}---where explicit feature engineering cannot easily capture the relevant patterns.

\subsection{Challenges and Limitations}

Several challenges were encountered during implementation. The Catch22 feature computation is sequential per time-series window, making it slower than vectorized raw feature extraction for large datasets. On TwoLeadECG (1,139 test samples), the feature extraction took approximately 2.67 seconds, which may be prohibitive for real-time applications.

The WESAD-like dataset is simulated, and our results may not generalize to real wearable sensor data, which exhibits more complex noise characteristics, motion artifacts, and inter-subject variability. Future work should validate the pipeline on the actual WESAD dataset.

Finally, the self-supervised embedding approach requires a separate training phase, adding complexity to the pipeline. The alignment between the embedding training window and the feature matrix required careful engineering to avoid off-by-one errors.

\section{Conclusion}

This project successfully implemented and evaluated a comprehensive, multi-task feature engineering framework for time-series data. Our key findings are:

\begin{enumerate}
    \item \textbf{Catch22 + LightGBM is a strong, efficient baseline} for time-series classification, matching or exceeding raw feature performance on 3 of 5 UCR datasets while compressing the feature space from 150+ time steps to just 22 interpretable features.
    \item \textbf{Lag and Rolling Statistics are the most effective features for forecasting}, reducing MSE by 82\% over the raw lag-1 baseline on ETTh1. Self-supervised embeddings did not provide additional benefit for this dataset.
    \item \textbf{SHAP-guided pruning is a reliable, principled dimensionality reduction method}, consistently achieving 40--45\% feature reduction with zero accuracy loss across all three task types.
    \item \textbf{Cross-domain generalization of global features is limited}: Catch22 fails on datasets where the class boundary is defined by local temporal patterns rather than global statistical properties.
\end{enumerate}

Future work will focus on three directions. First, integrating Large Language Models (LLMs) to automatically suggest domain-specific features based on dataset metadata prior to empirical evaluation. Second, developing ultra-lightweight Catch22 variants suitable for real-time deployment on IoT edge devices. Third, validating the full pipeline on the real WESAD wearable stress dataset to assess the practical utility of SHAP-guided anomaly detection for clinical applications.
